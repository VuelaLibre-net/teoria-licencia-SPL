# Política de seguridad

Este repositorio contiene principalmente contenido editorial, configuración de Quarto, scripts de compilación y automatización de GitHub. Aun así, pueden existir problemas de seguridad relacionados con la cadena de generación, los workflows, dependencias, artefactos publicados o permisos del repositorio.

## Reportar una vulnerabilidad

No abras una issue pública para vulnerabilidades o problemas que puedan comprometer el repositorio, los colaboradores o los usuarios.

Escribe en privado a:

```text
info@vuelalibre.net
```

Incluye, si puedes:

- Descripción clara del problema.
- Pasos para reproducirlo.
- Impacto posible.
- Ficheros, workflows o comandos afectados.
- Cualquier mitigación sugerida.

## Qué reportar por esta vía

- Exposición accidental de secretos o credenciales.
- Riesgos en GitHub Actions o scripts de compilación.
- Dependencias o herramientas comprometidas.
- Artefactos publicados que puedan inducir a descargar contenido malicioso.
- Cualquier fallo que requiera coordinación privada antes de hacerlo público.

## Qué puede ir en una issue pública

Las erratas, errores técnicos del temario, problemas de formato, cambios normativos y propuestas editoriales deben abrirse con el formulario de issues, salvo que incluyan información sensible.

## Versiones soportadas

Se revisan problemas de seguridad que afecten a la rama principal del repositorio y a los entregables publicados más recientes.

## Divulgación responsable

Intentaremos responder lo antes posible, confirmar la recepción del reporte y coordinar una corrección antes de publicar detalles. Por favor, no difundas públicamente la vulnerabilidad hasta que exista una mitigación o hayamos acordado un plazo razonable.
